YAML语法规则

发表于 2020-10-19 更新于 2020-10-21
本文字数： 966 阅读时长 ≈ 1 分钟

什么是YAML

  YML是专门用来写配置文件的语言，非常简洁和强大，远比 JSON 格式方便，它实质上是一种通用的数据串行化格式。

YML的语法规则

  • 大小写敏感
  • 使用缩进表示层级关系
  • 缩进时不允许使用TAB键，只允许使用空格
  • 缩进的空格数目不重要，只要相同层级的元素对应上即可

/#表示注释，从这个字符一直到行尾，都会被解析器忽略。

YAML支持三种数据结构类型

  • 对象：键值对的集合，又称为映射（mapping）/ 哈希（hashes） / 字典（dictionary）
  • 数组：一组按次序排列的值，又称为序列（sequence） / 列表（list）
  • 纯量（scalars）：单个的、不可再分的值

对象

  对象的一组键值对，使用冒号结构表示。格式为key: value。冒号后面要加一个空格：
animal: pets
转为javascript如下 

{ animal: 'pets' }

Yaml也允许另一种写法，将所有键值对都写成一个行内对象
hash：{name: Steve,foo: bar}
转为JavaScript如下
{ hash: { name: 'Steve', foo: 'bar' } }

数组

  一组连词线开头的行，构成一个数组。
-Cat
-Dog
-Fish
转为JavaScript如下
[ 'Cat', 'Dog', 'Fish' ]
数据结构的子成员是一个数组，则可以在下面缩进一格。使用一个短横线加一个空格代表一个数组项：
数组也可以采用行内表示法
animal: [cat,Dog]
较复杂的例子如下
companies:
一
id: 1
name: company1
price: 100
一
id: 2
name:company2
price:300
其等同于companies[{ id:1,name:company1,price:100},{id:2,name:company2,price:300}]
常量就不在此做介绍了，很少用到

注意事项

… 和—配合使用，在一个配置文件中代表一个文件的结束
!! YAML中使用!!做类型强行转换
>在字符串中折叠换行，| 保留换行符，这两个符号是YAML中字符串经常使用的符号
